## 技术栈演进

### 第一版：Quartz 5（已废弃）

Quartz 是为 Obsidian 设计的静态网站生成器，最初选择它的原因：
- 原生支持 `[[wikilink]]`
- 支持图谱视图和反向链接
- 社区活跃

**遇到的问题：**
- GitHub Actions 构建时需从 GitHub 拉取 46 个社区插件，网络不稳定时频繁失败
- SPA 模式导致静态 HTML 文件（如 AI 晨报 HTML 日报）被 Service Worker 拦截，持续出现 404
- 构建耗时 4-5 秒
- 自定义 CSS 每次 `git reset` 后丢失，需反复恢复

### 第二版：MkDocs Material（当前）

MkDocs Material 是业界最流行的 Markdown 文档工具，切换原因：
- **0.4 秒极速构建**（Quartz 的 1/10）
- Python 生态，3 个 pip 包即可，无 Git 插件依赖
- 自带顶级 Material Design 主题，无需手写 CSS
- 不存在 SPA 404 问题，所有页面直接加载
- GitHub Actions 部署仅 3 行命令

## mkdocs.yml 关键配置

```yaml
site_name: RFAUnion
site_url: https://rfaunion.github.io/obsidian/

theme:
  name: material
  language: zh                          # 中文界面
  palette:
    - scheme: default                   # 亮色：琥珀主题
      primary: amber
      accent: deep orange
    - scheme: slate                     # 暗色：slate 主题
      primary: amber
      accent: deep orange
  font:
    text: Noto Sans SC                  # 中文字体
    code: JetBrains Mono                # 代码字体
  features:
    - navigation.instant                # 即时导航
    - navigation.tabs                   # 顶部标签导航
    - search.suggest                    # 搜索建议
    - content.code.copy                 # 代码复制按钮

plugins:
  - search:
      lang: zh                          # 中文搜索
  - roamlinks                           # 支持 Obsidian [[wikilink]]
  - awesome-pages                       # 自动目录导航

docs_dir: content                       # 笔记源目录
site_dir: public                        # 构建输出目录
```

## GitHub Actions 自动部署

`.github/workflows/deploy.yml`：

```yaml
name: Deploy MkDocs Site
on:
  push:
    branches: [main]
  workflow_dispatch:

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.12"
      - run: pip install mkdocs mkdocs-material \
               mkdocs-awesome-pages-plugin mkdocs-roamlinks-plugin
      - run: mkdocs build
      - uses: peaceiris/actions-gh-pages@v4
        with:
          publish_dir: ./public
          force_orphan: true
```

**触发条件：** 推送到 `main` 分支自动构建，或手动触发。

## 本地开发流程

### 环境准备

```bash
# 安装 Python 依赖
pip install mkdocs mkdocs-material mkdocs-awesome-pages-plugin mkdocs-roamlinks-plugin
```

### 本地预览

```bash
cd obsidian-mkdocs
mkdocs serve
# 浏览器打开 http://localhost:8000
```

### 构建

```bash
mkdocs build
# 输出在 public/ 目录
```

### 发布流程

```bash
# 1. 在 Obsidian 中写笔记（content/ 目录）
# 2. 本地预览确认
mkdocs serve

# 3. 提交并推送
git add content/
git commit -m "📝 更新笔记"
git push origin main

# GitHub Actions 自动构建部署，无需手动操作
```

## 已知问题与解决

### HTML 日报文件部署

AI 晨报 HTML 文件需在 `mkdocs build` 后手动复制到 `public/ai-report/`，因为 MkDocs 不会处理 `content/` 下的 `.html` 文件。

**当前方案：** 构建后手动复制（由助手在部署时处理）。

**改进方向：** 可在 GitHub Actions 的 `deploy.yml` 中添加步骤：
```yaml
- run: |
    mkdir -p public/ai-report
    cp content/AI/AI晨报/html/*.html public/ai-report/
```

### GitHub Actions 并发冲突

多个 workflow 同时运行时，gh-pages 分支会因锁冲突失败。GitHub Actions 的 `concurrency` 配置可解决：

```yaml
concurrency:
  group: pages
  cancel-in-progress: true
```

### Obsidian Sync 频繁推送

Obsidian Sync 会频繁推送 `.obsidian/sync-stats.json` 到 main 分支，导致其他推送被锁。建议在 `.gitignore` 中忽略或配置 Sync 降低频率。
